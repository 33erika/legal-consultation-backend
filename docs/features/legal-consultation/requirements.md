# Requirements: 法务部法律咨询系统

## Overview

法务部法律咨询系统用于解决钉钉沟通效率低、数据无法复用、文件易丢失的问题，实现法律咨询工作的线上化、规范化、可追溯。系统支持三类业务：法律咨询（简单问题/复杂问题）、合同模板管理（拟写/更新含审批流程）、审批通知。

## User Stories

### Story 1: 用户认证与权限管理

As a 系统管理员, I want 管理用户账号和权限, so that 确保系统安全访问

**Acceptance Criteria:**

- [ ] AC1: 用户注册与登录
  - **Given**: 用户访问系统首页
  - **When**: 用户输入工号/密码并点击登录
  - **Then**: 系统验证凭证，成功后跳转首页，失败显示「工号或密码错误」

- [ ] AC2: 用户角色定义
  - **Given**: 系统已配置用户数据
  - **When**: 系统加载用户信息
  - **Then**: 用户被分配角色（普通员工/法务人员/法务主管/系统管理员），不同角色看到不同菜单

- [ ] AC3: 权限控制
  - **Given**: 普通员工已登录
  - **When**: 普通员工尝试访问法务人员专属页面（如待办列表）
  - **Then**: 系统返回 403 禁止访问，显示「无权限访问此页面」

- [ ] AC4: 登出功能
  - **Given**: 用户已登录系统
  - **When**: 用户点击「退出登录」
  - **Then**: 清除 session/token，跳转至登录页

### Story 2: 发起法律咨询（员工）

As a 普通员工, I want 提交法律咨询请求, so that 获得法务部的专业回复

**Acceptance Criteria:**

- [ ] AC1: 提交简单问题咨询
  - **Given**: 普通员工已登录系统
  - **When**: 员工选择「咨询类型」为「简单问题」，填写问题描述，点击「提交」
  - **Then**:
    - 创建咨询记录，状态为「待处理」
    - 返回 HTTP 201，响应体 `{ success: true, data: { id: string, ticket_no: string, status: "pending" } }`
    - 显示「提交成功」提示
    - 通过钉钉机器人通知法务人员

- [ ] AC2: 提交复杂问题咨询（需选择分类）
  - **Given**: 普通员工已登录系统
  - **When**: 员工选择「咨询类型」为「复杂问题」，选择子分类（纠纷咨询/合同/劳动/客诉/知识产权/其他），填写详细描述和附件，点击「提交」
  - **Then**:
    - 创建咨询记录，状态为「待处理」，分类为所选类型
    - 上传附件至服务器，生成唯一文件路径
    - 返回 HTTP 201，响应体包含咨询 ID 和工单号
    - 显示「提交成功」提示
    - 通过钉钉机器人通知法务人员

- [ ] AC3: 提交时附件上传
  - **Given**: 普通员工已登录，正在填写咨询表单
  - **When**: 员工选择文件（支持 pdf/doc/docx/xlsx/jpg/png，单文件 ≤ 20MB，最多 5 个附件）并上传
  - **Then**:
    - 文件上传至服务器 `/uploads/consultations/{consultation_id}/`
    - 返回文件上传结果 `{ success: true, data: { file_id: string, file_name: string, file_size: number, file_url: string } }`
    - 界面显示已上传文件列表，可删除

- [ ] AC4: 咨询列表查看（员工视角）
  - **Given**: 普通员工已登录，有 10 条历史咨询记录
  - **When**: 员工进入「我的咨询」页面
  - **Then**:
    - 显示咨询列表，按时间倒序排列
    - 每条记录显示：工单号、咨询类型、状态、提交时间
    - 支持按状态筛选（全部/待处理/处理中/已完成）
    - 支持按日期范围筛选
    - 分页显示，每页 20 条

- [ ] AC5: 咨询详情查看
  - **Given**: 员工有 1 条处理中的咨询记录
  - **When**: 员工点击查看该咨询详情
  - **Then**:
    - 显示完整咨询信息：工单号、类型、分类、描述、附件列表、提交时间
    - 显示法务人员回复内容（时间倒序排列）
    - 显示当前状态和历史操作记录

### Story 3: 处理法律咨询（法务人员）

As a 法务人员, I want 处理员工提交的法律咨询, so that 提供专业的法律支持

**Acceptance Criteria:**

- [ ] AC1: 查看待处理咨询列表
  - **Given**: 法务人员已登录系统，有 5 条待处理咨询
  - **When**: 法务人员进入「待办咨询」页面
  - **Then**:
    - 显示待处理咨询列表，按提交时间正序排列（先到先处理）
    - 每条显示：工单号、类型、员工姓名、部门、提交时间、紧急程度
    - 显示待办数量角标

- [ ] AC2: 接受咨询任务
  - **Given**: 法务人员查看某条待处理咨询详情
  - **When**: 法务人员点击「接受任务」
  - **Then**:
    - 咨询状态更新为「处理中」
    - 记录处理人（当前法务人员）
    - 记录接受时间
    - 通过钉钉机器人通知咨询人「您的咨询已被接受，处理人：XXX」

- [ ] AC3: 回复咨询
  - **Given**: 法务人员已接受某条咨询，状态为「处理中」
  - **When**: 法务人员输入回复内容并点击「发送回复」
  - **Then**:
    - 创建回复记录，关联咨询 ID
    - 更新咨询的最后回复时间
    - 通过钉钉机器人通知咨询人
    - 返回 HTTP 201

- [ ] AC4: 补充附件
  - **Given**: 法务人员正在处理某条咨询
  - **When**: 法务人员上传附件并发送
  - **Then**:
    - 附件上传至 `/uploads/replies/{consultation_id}/`
    - 回复内容包含附件信息
    - 咨询人可查看和下载附件

- [ ] AC5: 标记咨询完成
  - **Given**: 法务人员已回复咨询，处理完成
  - **When**: 法务人员点击「标记完成」
  - **Then**:
    - 咨询状态更新为「已完成」
    - 记录完成时间
    - 通过钉钉机器人通知咨询人「您的咨询已处理完毕」
    - 咨询人可对本次服务进行评价（1-5 星）

- [ ] AC6: 处理记录全程留痕
  - **Given**: 某条咨询从提交到完成的完整流程
  - **When**: 查看该咨询的历史记录
  - **Then**:
    - 按时间顺序显示所有操作：提交、接受、每次回复、状态变更、完成
    - 每条记录包含：操作人、操作时间、操作类型、操作详情
    - 记录不可删除、不可修改

### Story 4: 合同模板管理（员工发起）

As a 普通员工, I want 申请合同模板拟写或更新, so that 获得合规的合同模板

**Acceptance Criteria:**

- [ ] AC1: 发起模板申请
  - **Given**: 普通员工已登录系统
  - **When**: 员工选择「申请类型」为「合同模板」，选择操作类型（拟写/更新），填写需求描述，选择合同类型（如：采购合同/劳动合同/租赁合同等），点击「提交」
  - **Then**:
    - 创建模板申请记录，状态为「草稿」
    - 返回 HTTP 201，响应体 `{ success: true, data: { id: string, status: "draft" } }`
    - 跳转到申请详情页，提示「请提交给上级审批」

- [ ] AC2: 提交审批
  - **Given**: 员工有 1 条草稿状态的模板申请
  - **When**: 员工点击「提交审批」
  - **Then**:
    - 申请状态更新为「待 L1 审批」
    - 记录提交时间
    - 通知申请人上级进行审批
    - 通过钉钉机器人通知审批人

### Story 5: 合同模板审批流程（4级审批）

As a 审批人（L1 申请人上级 / L2 法务拟写 / L3 上级确认）, I want 审批合同模板申请, so that 确保流程合规

**Acceptance Criteria:**

- [ ] AC1: L1 审批（申请人上级）
  - **Given**: 审批人登录系统有待 L1 审批的申请
  - **When**: 审批人查看申请详情，点击「同意」或「拒绝」
  - **Then**:
    - 若同意：状态更新为「待 L2 拟写」，通知法务人员进行拟写
    - 若拒绝：状态更新为「已拒绝」，通知申请人被拒绝及原因
    - 记录审批人、审批时间、审批意见
    - 通过钉钉机器人通知相关人

- [ ] AC2: L2 拟写（法务人员）
  - **Given**: 法务人员有待 L2 拟写的申请
  - **When**: 法务人员上传合同模板文件，填写模板说明，点击「提交」
  - **Then**:
    - 状态更新为「待 L3 确认」
    - 记录法务人员、提交时间、模板文件路径
    - 通知 L3 审批人（申请人上级）进行确认
    - 通过钉钉机器人通知审批人

- [ ] AC3: L3 确认（申请人上级）
  - **Given**: L3 审批人有待确认的申请
  - **When**: 审批人查看模板内容，点击「确认发布」或「退回修改」
  - **Then**:
    - 若确认发布：状态更新为「已发布」，模板正式生效
    - 若退回修改：状态更新为「待 L2 拟写」（重新拟写），通知法务人员
    - 记录审批意见
    - 通过钉钉机器人通知相关人

- [ ] AC4: 审批状态可视化
  - **Given**: 用户查看某条模板申请详情
  - **When**: 查看审批流程状态
  - **Then**:
    - 显示流程图：申请人提需 → L1 审批 → L2 拟写 → L3 确认 → 发布
    - 明确标注当前所处步骤
    - 显示每一步的审批人、审批时间、审批结果

- [ ] AC5: 审批历史记录
  - **Given**: 某条模板申请已完成审批流程
  - **When**: 查看该申请的历史审批记录
  - **Then**:
    - 显示完整的审批链：申请人提交 → L1 审批 → L2 拟写 → L3 确认
    - 每条记录包含：审批人姓名、角色、审批时间、审批结果、审批意见
    - 记录不可删除、不可修改

### Story 6: 合同模板库管理

As a 法务人员, I want 管理合同模板库, so that 提供标准化、可复用的合同模板

**Acceptance Criteria:**

- [ ] AC1: 查看已发布模板列表
  - **Given**: 法务人员已登录系统，库中有 20 份已发布模板
  - **When**: 法务人员进入「模板库」页面
  - **Then**:
    - 显示已发布模板列表，按发布时间倒序
    - 每条显示：模板名称、合同类型、版本号、发布时间、发布人
    - 支持按合同类型筛选
    - 支持关键词搜索
    - 分页显示，每页 20 条

- [ ] AC2: 模板详情与下载
  - **Given**: 模板库中有已发布模板
  - **When**: 用户点击查看模板详情
  - **Then**:
    - 显示模板信息：名称、类型、版本、描述、发布说明
    - 支持预览模板内容（支持 pdf/doc/docx）
    - 支持下载模板文件
    - 显示模板使用记录（被哪些申请引用）

- [ ] AC3: 模板版本管理
  - **Given**: 某份模板已发布 v1.0
  - **When**: 法务人员发起更新申请，新版本通过审批发布
  - **Then**:
    - 模板版本更新为 v1.1
    - 保留历史版本记录，可查看历史版本
    - 历史版本标记为「已归档」
    - 归档版本可下载但不可设为默认

- [ ] AC4: 模板禁用/启用
  - **Given**: 法务人员需要暂时停用某份模板
  - **When**: 法务人员点击「禁用」
  - **Then**:
    - 模板状态更新为「已禁用」
    - 禁用后模板不在列表中显示（除非勾选「显示已禁用」）
    - 已使用该模板的申请不受影响

### Story 7: 钉钉通知集成

As a 系统, I want 发送钉钉机器人通知, so that 相关人员及时获知重要事项

**Acceptance Criteria:**

- [ ] AC1: 新咨询提交通知
  - **Given**: 员工提交了新咨询
  - **When**: 系统处理提交请求完成
  - **Then**:
    - 调用钉钉机器人 Webhook，发送通知
    - 通知内容：「📋 新的法律咨询 | 工单号：{ticket_no} | 类型：{type} | 提交人：{name}」
    - 通知链接：「点击查看」→ 跳转系统详情页

- [ ] AC2: 咨询状态变更通知
  - **Given**: 咨询状态发生变更（被接受/被回复/已完成）
  - **When**: 系统更新咨询状态完成
  - **Then**:
    - 调用钉钉机器人 Webhook 通知咨询人
    - 通知内容包含：状态变更说明、操作人、处理时间
    - 通知链接指向咨询详情页

- [ ] AC3: 审批流程通知
  - **Given**: 模板申请进入下一审批节点
  - **When**: 系统更新申请状态
  - **Then**:
    - 调用钉钉机器人 Webhook 通知当前审批人
    - 通知内容：「📝 合同模板申请待审批 | 申请人：{name} | 操作类型：{operation} | 类型：{template_type}」
    - 通知链接指向审批详情页

- [ ] AC4: 审批结果通知
  - **Given**: 模板申请审批完成（通过/拒绝）
  - **When**: 系统更新申请状态
  - **Then**:
    - 调用钉钉机器人 Webhook 通知申请人
    - 通知内容包含：审批结果、审批人意见
    - 通知链接指向申请详情页

- [ ] AC5: 通知配置管理
  - **Given**: 系统管理员需要配置钉钉通知
  - **When**: 管理员进入「系统设置」→「通知设置」
  - **Then**:
    - 可配置 Webhook URL
    - 可配置通知开关（开启/关闭各类型通知）
    - 可测试通知连通性（发送测试消息）

### Story 8: 数据统计与报表

As a 法务主管, I want 查看咨询统计数据, so that 了解法务部工作负载和效率

**Acceptance Criteria:**

- [ ] AC1: 咨询统计概览
  - **Given**: 系统中有历史咨询数据
  - **When**: 法务主管进入「数据统计」页面
  - **Then**:
    - 显示统计卡片：本周新增咨询数、本月新增咨询数、待处理数、本周完成数
    - 显示趋势图：近 30 天咨询数量趋势
    - 显示分类占比图：各类型咨询占比

- [ ] AC2: 咨询类型分布
  - **Given**: 系统中有历史数据
  - **When**: 查看咨询类型分布
  - **Then**:
    - 显示饼图：简单问题/复杂问题（纠纷咨询/合同/劳动/客诉/知识产权/其他）
    - 支持切换时间范围（本周/本月/本季度/本年/自定义）

- [ ] AC3: 处理效率统计
  - **Given**: 系统中有已完成咨询数据
  - **When**: 查看处理效率统计
  - **Then**:
    - 显示平均处理时长（从提交到完成）
    - 显示法务人员工作量排行
    - 支持按法务人员筛选

- [ ] AC4: 导出统计报表
  - **Given**: 法务主管查看统计数据
  - **When**: 点击「导出报表」
  - **Then**:
    - 导出 Excel 文件，包含：咨询明细、处理时长统计、类型分布
    - 文件名格式：`法务咨询统计_{YYYYMMDD}.xlsx`

### Story 9: 搜索与历史案例复用

As a 法务人员, I want 搜索历史咨询和案例, so that 快速复用已有解决方案

**Acceptance Criteria:**

- [ ] AC1: 全文搜索
  - **Given**: 系统中有 1000 条历史咨询
  - **When**: 法务人员在搜索框输入关键词
  - **Then**:
    - 支持模糊搜索（标题、描述、回复内容）
    - 支持按类型、状态、日期范围筛选
    - 搜索结果高亮关键词
    - 返回相关度排序结果

- [ ] AC2: 相似问题推荐
  - **Given**: 员工提交新咨询时输入了问题描述
  - **When**: 输入过程中
  - **Then**:
    - 系统实时分析输入内容
    - 显示相似度 > 80% 的历史咨询标题
    - 点击可查看历史咨询详情和解决方案
    - 帮助员工自助解决或提供参考

- [ ] AC3: 案例收藏
  - **Given**: 法务人员查看历史咨询详情
  - **When**: 点击「收藏」
  - **Then**:
    - 将该咨询标记为案例
    - 可在「我的收藏」中快速查看
    - 收藏时可选添加标签（如：劳动纠纷/合同审核）

## Constraints

- **性能要求**：
  - 页面加载时间 < 3 秒（P95）
  - API 响应时间 < 2 秒（P95）
  - 支持 100 用户同时在线

- **安全要求**：
  - 所有 API 需认证访问
  - 密码加密存储（bcrypt）
  - 敏感操作需记录日志
  - 文件上传需校验文件类型和大小

- **技术约束**：
  - 前端：Next.js + Shadcn UI
  - 后端：Go
  - 数据库：PostgreSQL
  - 配置：YAML 文件
  - 附件存储：本地文件系统

- **部署约束**：
  - 单台服务器部署
  - 无 Docker 环境（可选）

## Out of Scope

- 钉钉 SSO 单点登录（当前仅支持用户名密码登录）
- 多租户支持
- 移动端 App
- 防作弊/安全审计功能
- 合同模板的在线编辑（仅支持文件上传）
- 与外部法律数据库集成

## Assumptions

以下假设已与用户确认：
- [x] 使用 PostgreSQL 数据库（替代 SQLite）
- [x] 用户认证使用用户名密码，不集成钉钉 SSO
- [x] 附件存储在服务器本地文件系统
- [x] 审批流程使用状态机模式实现
- [x] 新系统从零开始，不迁移历史数据
- [x] 通知使用钉钉机器人 Webhook
- [x] 单台服务器部署，无高并发需求
