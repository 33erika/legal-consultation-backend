# Requirements: 法务部法律咨询系统

## Overview

法务部法律咨询系统用于解决钉钉沟通效率低、数据无法复用、文件易丢失的问题，实现法律咨询工作的线上化、规范化、可追溯。系统支持两类核心业务：
1. **法律咨询**：员工提问 → 法务接单 → 法务回复 → 结案
2. **合同模板管理**：员工提需 → 业务主管审批 → 法务拟写 → 法务负责人审核 → 发布生效

## 用户角色

| 角色 | 职责 |
|------|------|
| **普通员工** | 全体员工，发起咨询或合同提需 |
| **业务主管** | 申请人直接上级，负责合同类需求的成本/必要性把关 |
| **法务专员** | 处理日常咨询、拟写合同草案 |
| **法务负责人** | 负责合同母本的最终发布审批 |
| **系统管理员** | 管理系统基础数据 |

## 核心页面设计

### 智能分流页（员工入口）

As a 普通员工, I want 通过大卡片选择业务类型, so that 减少误操作，快速进入对应流程

**Acceptance Criteria:**

- [ ] AC1: 分流页展示
  - **Given**: 员工登录系统或点击「我要咨询」
  - **When**: 进入业务选择页面
  - **Then**:
    - 显示两个大卡片：「法律咨询」「合同模板提需」
    - 卡片尺寸醒目（高度 ≥ 150px），带图标和简短说明
    - 点击后进入对应表单页面
    - 视觉上清晰区分两个业务入口

- [ ] AC2: 卡片内容
  - **Given**: 员工查看分流页
  - **When**: 查看卡片
  - **Then**:
    - 「法律咨询」卡片：图标 + 标题「法律咨询」+ 说明「提交法律问题咨询，获取法务专业回复」
    - 「合同模板提需」卡片：图标 + 标题「合同模板提需」+ 说明「申请新增或更新合同模板」

### 法务工作台

As a 法务专员, I want 查看和处理待办工作, so that 高效管理咨询和模板任务

**Acceptance Criteria:**

- [ ] AC1: 工作台看板 - 咨询统计
  - **Given**: 法务专员登录系统
  - **When**: 进入工作台
  - **Then**:
    - 显示今日看板统计：
      - 今日新增咨询数（带数量显示）
      - 今日已回复数（带数量显示）
      - 待处理咨询数（带数量显示，超过0显示警告色）
      - 本周已结案数（带数量显示）
    - 看板数据实时更新

- [ ] AC2: 工作台看板 - 模板统计
  - **Given**: 法务专员登录系统
  - **When**: 进入工作台
  - **Then**:
    - 显示模板相关统计：
      - 待拟写模板数（带数量显示）
      - 待审核模板数（带数量显示）
      - 今日新增申请数（带数量显示）

- [ ] AC3: 快捷入口
  - **Given**: 法务专员在工作台
  - **When**: 查看工作台
  - **Then**:
    - 显示快捷入口：「咨询池」「我的待办」「待拟写」「待审核」
    - 每个入口显示当前待办数量

- [ ] AC4: 最近处理记录
  - **Given**: 法务专员查看工作台
  - **When**: 查看最近处理
  - **Then**:
    - 显示最近处理记录（最近 5 条）
    - 每条显示：工单号、问题标题、操作类型、操作时间

- [ ] AC5: 咨询池
  - **Given**: 法务专员进入「咨询池」
  - **When**: 查看待处理咨询
  - **Then**:
    - 展示所有待处理咨询（未接单的咨询）
    - 按紧急程度 + 提交时间排序（非常紧急优先）
    - 每条显示：工单号、问题标题、员工姓名、部门、紧急程度、提交时间
    - 支持按紧急程度筛选
    - 法务点击「接单」后，该咨询进入个人待办

- [ ] AC6: 个人待办
  - **Given**: 法务专员已接单的咨询
  - **When**: 进入「我的待办」
  - **Then**:
    - 显示本人已接单但未结案的咨询
    - 每条显示：工单号、问题标题、员工姓名、紧急程度、接单时间、距今时长
    - 支持按状态筛选（处理中/已回复/待补充）
    - 超时提醒（超过 24 小时未结案显示警告色）

## User Stories

### Story 1: 用户认证与权限管理

As a 系统管理员, I want 管理用户账号和权限, so that 确保系统安全访问

**Acceptance Criteria:**

- [ ] AC1: 用户登录
  - **Given**: 用户访问系统登录页
  - **When**: 用户输入工号/密码并点击登录
  - **Then**: 系统验证凭证，成功后跳转首页，失败显示「工号或密码错误」，返回 HTTP 401

- [ ] AC2: 角色自动识别
  - **Given**: 用户登录成功
  - **When**: 系统加载用户信息
  - **Then**: 根据用户绑定的角色（普通员工/业务主管/法务专员/法务负责人/系统管理员），加载对应菜单和权限

- [ ] AC3: 权限控制
  - **Given**: 普通员工已登录
  - **When**: 普通员工尝试访问法务专员专属页面（如咨询池、模板拟写）
  - **Then**: 系统返回 HTTP 403，显示「无权限访问此页面」

- [ ] AC4: 登出功能
  - **Given**: 用户已登录系统
  - **When**: 用户点击「退出登录」
  - **Then**: 清除 session/token，跳转至登录页

### Story 2: 法律咨询 - 发起咨询（员工）

As a 普通员工, I want 提交法律咨询请求, so that 获得法务部的专业回复

**Acceptance Criteria:**

- [ ] AC1: 提交法律咨询
  - **Given**: 普通员工已登录系统，通过分流页进入法律咨询表单
  - **When**: 员工填写以下信息并点击提交：
    - 问题标题（必填，≤ 100 字，简要描述问题）
    - 问题描述（必填，≤ 5000 字，详细说明背景、情况）
    - 紧急程度（必填）：一般 / 紧急 / 非常紧急
    - 相关附件（选填，最多 10 个，单文件 ≤ 50MB，支持 pdf/doc/docx/xlsx/jpg/png）
  - **Then**:
    - 创建咨询记录，状态为「待处理」
    - 上传附件至服务器 `/uploads/consultations/{consultation_id}/`
    - 返回 HTTP 201，响应体包含咨询 ID 和工单号（格式：`CONS-{YYYYMMDD}-{序号}`）
    - 显示「提交成功」提示
    - 通过钉钉机器人通知法务专员：「📋 您有一个新的法律咨询待处理 | 工单号：{ticket_no}」

- [ ] AC2: 咨询类型引导 - 客诉类
  - **Given**: 员工选择咨询类型为「客诉」或输入内容触发客诉关键词
  - **When**: 员工填写咨询表单
  - **Then**:
    - 系统自动提示填写引导：
      - 「是否已向商家/平台投诉？」+ 是/否 选择
      - 「是否有工单号/投诉单号？」+ 文本输入
      - 「投诉类型」+ 下拉选择（向商家投诉/向平台投诉/向工商投诉/向消协投诉/其他）
      - 「目前诉求」+ 文本输入（≤ 500 字）
      - 「相关证据」+ 附件上传提示（建议上传：聊天记录截图/商品照片/发票/合同等）
    - 提示信息在表单中以气泡提示形式展示
    - 填写项为建议性，不强制必填

- [ ] AC3: 咨询类型引导 - 合同类
  - **Given**: 员工选择咨询类型为「合同」或输入内容触发合同关键词
  - **When**: 员工填写咨询表单
  - **Then**:
    - 系统自动提示填写引导：
      - 「合同类型」+ 下拉选择（采购合同/销售合同/租赁合同/服务合同/劳动合同/其他）
      - 「合同相对方」+ 文本输入（公司名称或个人姓名）
      - 「合同金额」+ 数字输入（单位：元）
      - 「合同是否已签署」+ 是/否 选择
      - 「主要争议点」+ 文本输入（≤ 500 字）
      - 「相关合同附件」+ 附件上传提示（建议上传合同扫描件）
    - 提示信息在表单中以气泡提示形式展示

- [ ] AC4: 咨询类型引导 - 劳动类
  - **Given**: 员工选择咨询类型为「劳动」或输入内容触发劳动关键词
  - **When**: 员工填写咨询表单
  - **Then**:
    - 系统自动提示填写引导：
      - 「员工身份」+ 下拉选择（在职员工/离职员工/试用期员工/实习生）
      - 「问题类型」+ 下拉选择（工资/赔偿金/加班费/社保/合同解除/其他）
      - 「涉及人数」+ 数字输入
      - 「是否已发送书面通知」+ 是/否 选择
      - 「相关证据」+ 附件上传提示（建议上传劳动合同/工资流水/通知函/聊天记录等）
    - 提示信息在表单中以气泡提示形式展示

- [ ] AC5: 咨询类型引导 - 知识产权类
  - **Given**: 员工选择咨询类型为「知识产权」或输入内容触发知识产权关键词
  - **When**: 员工填写咨询表单
  - **Then**:
    - 系统自动提示填写引导：
      - 「知识产权类型」+ 下拉选择（商标/专利/著作权/商业秘密/其他）
      - 「是否已进行登记」+ 是/否 选择
      - 「登记号/证书号」+ 文本输入
      - 「侵权形式」+ 文本输入（≤ 500 字）
      - 「侵权证据」+ 附件上传提示
    - 提示信息在表单中以气泡提示形式展示

- [ ] AC6: 咨询类型引导 - 纠纷咨询类
  - **Given**: 员工选择咨询类型为「纠纷咨询」或输入内容触发纠纷关键词
  - **When**: 员工填写咨询表单
  - **Then**:
    - 系统自动提示填写引导：
      - 「纠纷阶段」+ 下拉选择（尚未起诉/准备起诉/已起诉未开庭/已开庭/已判决/执行阶段）
      - 「对方当事人」+ 文本输入
      - 「纠纷金额」+ 数字输入（单位：元）
      - 「是否已有律师」+ 是/否 选择
      - 「案件概要」+ 文本输入（≤ 500 字）
      - 「相关材料」+ 附件上传提示（建议上传起诉状/判决书/证据材料等）
    - 提示信息在表单中以气泡提示形式展示

- [ ] AC7: 员工补充资料
  - **Given**: 咨询状态为「已回复」，法务要求补充资料
  - **When**: 员工上传补充资料并提交
  - **Then**:
    - 咨询状态保持不变（仍为「已回复」），不退回待处理
    - 添加员工回复记录，包含补充说明和附件
    - 通知法务专员有新补充内容
    - 返回 HTTP 201

- [ ] AC8: 查看我的咨询列表
  - **Given**: 普通员工已登录，有历史咨询记录
  - **When**: 员工进入「我的咨询」页面
  - **Then**:
    - 显示咨询列表，按时间倒序排列
    - 每条显示：工单号、问题标题、状态、提交时间
    - 支持按状态筛选（全部/待处理/处理中/已回复/已结案）
    - 支持按日期范围筛选
    - 支持关键词搜索（标题、工单号）
    - 分页显示，每页 20 条

- [ ] AC9: 查看咨询详情
  - **Given**: 员工有 1 条处理中的咨询记录
  - **When**: 员工点击查看该咨询详情
  - **Then**:
    - 显示完整信息：工单号、问题标题、问题描述、紧急程度、状态、提交时间
    - 显示附件列表（可下载）
    - 显示对话记录（法务回复 + 员工补充，按时间正序）
    - 若法务要求补充资料，显示「请补充以下资料」提示
    - 显示操作按钮（补充资料/评价/关闭）

- [ ] AC10: 相似问题推荐
  - **Given**: 员工提交新咨询时输入问题描述
  - **When**: 输入过程中（输入 ≥ 20 字时）
  - **Then**:
    - 系统实时分析输入内容
    - 显示相似度 > 70% 的历史咨询标题（最多显示 5 条）
    - 点击可查看历史咨询详情和解决方案

### Story 3: 法律咨询 - 处理咨询（法务专员）

As a 法务专员, I want 处理员工提交的法律咨询, so that 提供专业的法律支持

**Acceptance Criteria:**

- [ ] AC1: 接单咨询
  - **Given**: 法务专员查看咨询池中某条待处理咨询
  - **When**: 法务专员点击「接单」
  - **Then**:
    - 咨询状态更新为「处理中」
    - 记录处理人（当前法务专员）、接单时间
    - 通过钉钉机器人通知咨询人：「🔔 您的咨询已被接受 | 处理人：{姓名}」
    - 该咨询从咨询池移除，进入法务个人待办
    - 返回 HTTP 200

- [ ] AC2: 接单时内部标记分类
  - **Given**: 法务专员接单时
  - **When**: 法务专员点击「接单」并设置内部分类
  - **Then**:
    - 系统弹出分类选择框
    - 分类选项：简单问题（默认）/ 复杂问题（含子分类：纠纷咨询/合同/劳动/客诉/知识产权/其他）
    - 若选复杂问题，需选择具体子分类
    - 分类仅内部可见，不展示给员工
    - 保存后进入处理状态

- [ ] AC3: 变更处理人
  - **Given**: 法务专员正在处理某条咨询，或需要将咨询转交给其他法务
  - **When**: 法务专员点击「变更处理人」，选择其他法务专员
  - **Then**:
    - 弹出处理人选择框，显示所有在职法务专员列表
    - 可选择新处理人并填写变更原因（选填）
    - 确认后，更新咨询处理人
    - 通知原处理人：「您的咨询 {ticket_no} 已转交给 {新处理人}」
    - 通知新处理人：「您有一个新转入的咨询待处理 | 工单号：{ticket_no}」
    - 记录变更历史（操作人、变更时间、原处理人、新处理人、原因）
    - 仅法务负责人或原处理人可以发起变更

- [ ] AC4: 回复咨询
  - **Given**: 法务专员已接单某条咨询，状态为「处理中」
  - **When**: 法务专员输入回复内容，可选上传附件，点击「发送回复」
  - **Then**:
    - 创建回复记录，关联咨询 ID、回复人
    - 咨询状态更新为「已回复」
    - 记录回复时间
    - 通过钉钉机器人通知咨询人：「💬 您的咨询有新回复 | 点击查看详情」
    - 返回 HTTP 201

- [ ] AC5: 要求补充资料
  - **Given**: 法务专员正在处理某条咨询，需要更多信息
  - **When**: 法务专员点击「要求补充资料」，填写需要补充的内容说明
  - **Then**:
    - 咨询状态保持「已回复」（不退回待处理）
    - 添加系统消息「法务要求补充以下资料：{说明}」
    - 通过钉钉机器人通知咨询人

- [ ] AC6: 标记结案
  - **Given**: 法务专员已完整回复咨询，处理完成
  - **When**: 法务专员点击「标记结案」
  - **Then**:
    - 弹出结案确认框
    - 法务专员选择最终分类（简单问题/复杂问题-子类），默认与接单时一致
    - 若需修改分类，可在此调整
    - 咨询状态更新为「已结案」
    - 记录结案时间、最终分类
    - 通过钉钉机器人通知咨询人：「✅ 您的咨询已处理完毕，请对本次服务进行评价」
    - 咨询人可对本次服务进行评价（1-5 星）

- [ ] AC7: 处理记录全程留痕
  - **Given**: 某条咨询从提交到结案的完整流程
  - **When**: 查看该咨询的历史记录
  - **Then**:
    - 按时间顺序显示所有操作：
      1. 员工提交（时间、操作人、内容摘要）
      2. 法务接单（时间、操作人、内部分类）
      3. 处理人变更（如有）（时间、操作人、原处理人、新处理人、原因）
      4. 法务回复（时间、操作人、内容摘要）
      5. 员工补充（如有）
      6. 结案（时间、操作人、最终分类）
    - 每条记录包含：操作人姓名、角色、操作时间、操作类型
    - 记录不可删除、不可修改

- [ ] AC8: 咨询数据统计（法务专员）
  - **Given**: 法务专员查看工作量统计
  - **When**: 进入「我的工作统计」页面
  - **Then**:
    - 显示个人统计数据：本月处理咨询数、平均处理时长、待处理数、已结案数
    - 显示处理效率趋势图
    - 显示简单/复杂问题分类占比

### Story 4: 合同模板管理 - 发起申请（员工）

As a 普通员工, I want 申请合同模板拟写或更新, so that 获得合规的合同模板

**Acceptance Criteria:**

- [ ] AC1: 发起合同模板申请 - 基础表单
  - **Given**: 普通员工已登录系统，通过分流页进入合同模板提需
  - **When**: 员工填写以下基础信息：
    - 申请类型：新增模板 / 更新现有模板
    - 若为更新：选择现有模板（下拉选择）
    - 合同类型（必填）：采购合同 / 劳动合同 / 租赁合同 / 销售合同 / 保密协议 / 其他
    - 需求标题（必填，≤ 100 字）
  - **Then**: 进入下一步填写详细需求

- [ ] AC2: 发起合同模板申请 - 需求描述引导
  - **Given**: 员工填写合同模板需求
  - **When**: 员工填写需求场景描述
  - **Then**:
    - 系统显示结构化填写引导（气泡提示形式）：
      1. 「此模板适用的核心业务场景（业务类型、合作对象、核心事项）」

  - **Given**: 系统管理员需要配置各类别的填写提示
  - **When**: 进入「咨询引导配置」页面
  - **Then**:
    - 按咨询分类（客诉/合同/劳动/知识产权/纠纷咨询）分别配置填写引导

      - 引导问题列表（问题文本 + 问题类型 + 是否必填 + 示例）

    - 配置实时生效

- [ ] AC7: 系统参数配置
  - **Given**: 系统管理员已登录
  - **When**: 进入「系统设置」页面
  - **Then**:

    - 可配置工单号格式

## Constraints

- **性能要求**：
  - 页面加载时间 < 3 秒（P95）
  - API 响应时间 < 2 秒（P95）
  - 支持 100 用户同时在线

- **安全要求**：
  - 所有 API 需认证访问
  - 密码加密存储（bcrypt）
  - 敏感操作记录操作日志
  - 文件上传校验文件类型和大小

- **技术约束**：
  - 前端：Next.js + Shadcn UI
  - 后端：Go
  - 数据库：PostgreSQL
  - 配置：YAML 文件
  - 附件存储：本地文件系统

## Out of Scope

- 钉钉 SSO 单点登录（仅支持用户名密码登录）
- 多租户支持
- 移动端 App
- 防作弊/安全审计功能
- 合同模板在线编辑器（仅支持文件上传）
- 与外部法律数据库集成

## UI 设计规范

### 基础配色表

| 颜色角色 | 十六进制色值 | 视觉感受 | 用途 |
|----------|-------------|----------|------|
| 主色（Justice Blue） | #1A535C | 沉稳、专业、可信赖 | 导航栏、主要按钮、标题 |
| 辅助色（Tiffany Teal） | #4ECDC4 | 清爽、高效 | 成功状态、合同发布标签 |
| 警示色（Alert Orange） | #FF6B6B | 醒目、警示 | 驳回、超时预警、待办数字 |
| 背景色（Quiet Grey） | #F7F9FB | 干净、无干扰 | 页面大背景，减少视觉压力 |
| 文字黑（Deep Charcoal） | #2D3436 | 易读 | 正文内容 |

### 按钮状态设计

- **默认状态**：主色 #1A535C，白字
- **悬停状态 (Hover)**：颜色变浅 10%，带有微小阴影（给手指"按下去"的反馈感）
- **禁用状态 (Disabled)**：浅灰色，暗示流程未走到这一步

### 进度条设计（Stepper）

- 顶部永远显示清晰的进度条
- 进度节点：提交 → 法务处理中 → 已回复
- 让用户对"现在在哪"有掌控感

### 状态标签设计（Status Tags）

- **简单问题**：绿色细圆角标签
- **复杂问题**：紫色实心圆角标签（视觉权重更高）

### 空状态设计

- 当没有待办任务时，显示一张可爱的插画
- 配文字：「暂无任务，休息一下吧」
- 营造轻松的工作氛围

### 专注模式

- 法务在拟写合同模板时，可点击「全屏编辑」
- 隐藏所有侧边栏和通知
- 只留下文档编辑区
- 提供沉浸式编辑体验

### 智能分流页设计

- 两个大卡片横向排列
- 卡片尺寸：高度 ≥ 150px，宽度占据屏幕一半
- 卡片带图标（法律咨询用天平图标，合同模板用文件图标）
- 悬停时有轻微上浮和阴影效果
- 卡片间距：24px

### 法务工作台看板设计

- 统计卡片采用数字+标签的形式
- 待处理数字 > 0 时显示警示橙色
- 快捷入口采用图标+文字+数字角标形式
- 卡片布局：2行2列或横向排列
